# Example Tasks Reference

读取本文件用于判断典型用户请求应该选择什么模式、产出什么文件以及如何验收。

## System Review

用户请求：

> 用这些高数课件整理一份第 1 章复习讲义，适合期末复习。

模式：系统复习。

输出：

- `tex/main.tex` 或章节文件。
- 必要的 TikZ 图。
- 分层练习和完整解析。
- `review_notes/uncertainties.md`。

验收：

- 讲义自成一体，有定义、定理条件、典型例题、方法总结和易错点。
- 已完成困惑点覆盖检查：学生可能卡住的前置概念、条件作用、推导跳步、方法触发点和常见误用已映射到正文或例题；未确认项写入 `review_notes/confusion-audit.md`。
- LaTeX 能编译。
- 不确定公式和图像独立记录。

## Sprint Training

用户请求：

> 下周考试，只要连续和导数的高频题型，偏计算，少证明。

模式：冲刺训练。

输出：

- 高频考点清单。
- 题型模板和限时训练。
- 完整答案解析。

验收：

- 内容压缩，不展开过多背景。
- 即使压缩，也保留方法触发点、适用条件和高频误用提醒。
- 题目符合“偏计算、少证明”。
- 每题标注考点、难度和预计用时。

## Problem Bank

用户请求：

> 把这本习题册第 3 章做成题库，以后能按难度抽题。

模式：题库模式。

输出：

- `problem_bank/ch03-*.yml`。
- 必要时更新 `state.yml` 的标签集合。
- 扫描/OCR 不确定项写入 `review_notes/`。

验收：

- 每题有 id、考点、难度、题型、来源、题干、答案、解析和预计用时。
- 标签可用于后续抽题和组卷。

## Mock Exam

用户请求：

> 根据这三份往年题出一套 100 分模拟卷，附完整解析。

模式：试卷模式。

输出：

- 往年题风格摘要。
- `tex/exams/mock-01.tex`。
- 完整解析。

验收：

- 总分、时间、题型比例、难度梯度一致。
- 常考点覆盖合理。
- 风格贴近用户提供的往年题。

## Incremental Update

用户请求：

> 继续昨天的线代项目，把特征值这一章加进去。

模式：增量更新。

输出：

- 更新对应章节、习题、解析和题库条目。
- 更新 `state.yml`。
- 新增或解决 `review_notes/` 项。

验收：

- 不重写无关章节。
- 延续既有符号、编号、模板和题库标签。
- 新增章节完成困惑点审计，并更新 `review.confusion_audit` 或 `review_notes/confusion-audit.md`。
- 报告本次改动和编译结果。
