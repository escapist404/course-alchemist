# Source Processing Reference

读取本文件用于处理 PDF、扫描件、PPT、题目照片、往年题、图形重绘和核对清单。

## Material Paths

- 高质量文字 PDF：优先文本抽取，抽样截图核对公式和图形。
- 图像型 PDF 或扫描件：渲染逐页图片，同时 OCR，再用视觉理解核对公式、图形、版面层次和题目结构。
- PPT：提取文本和图片；保留页面顺序，但不要照搬碎片化排版。
- 题目照片：先转写题干，再标注不确定字符、公式和图形。
- 往年题：除转写题目外，还要总结题型比例、难度、常考点、计算量和表述风格。

## Mixed OCR Strategy

默认使用混合方案：

1. 渲染页面图片，保留版面、图形和题目结构。
2. 尝试文本抽取或 OCR，得到可复制文字和粗略公式。
3. 视觉核对公式、图表、板书层次、题号和选项。
4. 将定义、定理、例题、题干和解答重写为 LaTeX。
5. 按 `references/confusion-audit.md` 做困惑点审计，找出原材料一笔带过但学生可能卡住的定义展开、条件作用、推导跳步、图形直觉和方法触发点。
6. 对无法确认的信息写入 `review_notes/`。
7. 结合课程上下文补齐逻辑，使讲义自成一体。

不要把 OCR 输出直接当成讲义正文。OCR 是素材，不是成品。

实现入口：

- 扫描版课本优先使用 `scripts/process_scanned_textbook.py` 生成结构化素材。
- 组件、目录结构、page record schema 和验收标准见 `references/scanned-textbook-pipeline.md`。
- 默认输出写入 `sources/processed/<source-stem>/`；若用户提供课程项目目录，可用 `--output` 指向该项目的 `sources/processed/` 子目录。

## TikZ and Figures

- 数学函数图、几何图、普通物理受力图、电路图和坐标图优先用 TikZ/PGFPlots 重绘。
- 重绘时保持符号、坐标方向、单位和图例与正文一致。
- 若图像复杂、低清或不影响核心理解，可以保留裁剪原图，并在正文补解释。
- 对无法确认的箭头方向、标注、元件参数或坐标轴含义，写入核对清单。

## Figure Pass

对强依赖空间直觉的内容，生成讲义时要做一次专门的图示补强检查：

- 普通物理：电场线、磁感线、受力图、能量图、通量/环流路径、高斯面、等势面、坐标方向和边界条件应优先画图。
- 多元微积分：区域、曲线/曲面、法向、切向、投影区域、坐标变换、梯度/散度/旋度直观应优先画图。
- 线性代数：线性变换、基变换、子空间、投影、特征向量、二次型等几何意义应优先画图。
- 每个主要概念块至少判断一次“读者是否必须靠脑补空间关系”；若答案是是，应补 TikZ/PGFPlots 图或保留原图。
- 图示要服务推理：图注应说明图中箭头、法向、积分路径、对称面或坐标轴在公式中对应什么。
- 图内文字必须克制：只保留变量、方向、点名、面名等必要 label；解释性句子放在正文或 caption。
- 图中 label 不得与曲线、箭头、面域、坐标轴重叠；重绘后要渲染抽查。
- 图示关系必须正确：电场线、流线、等势线、轨迹、特征向量方向等不能出现与理论矛盾的交叉、方向或拓扑错误。

## Confusion Pass

生成或更新讲义时，必须在正文定稿前做一次困惑点补强检查：

- 对每个主要概念块列出 3 到 8 个学生可能追问的问题，优先覆盖“定义为什么这样写”“条件有什么用”“这一步怎么算出来”“什么时候能用这个方法”“和相近概念有什么区别”。
- 若原资料只有结论、公式或例题答案，不能照搬为讲义；要补上中间解释、方法触发点和至少一个边界提醒。
- 若某个困惑点能通过图示解决，应并入 Figure Pass，一起补 TikZ/PGFPlots 图或清晰图注。
- 若某个困惑点依赖源材料不清、图形方向不明或课程口径不同，写入 `review_notes/confusion-audit.md`，并在最终回复中说明。
- 定稿后反向检查：从困惑点清单逐项追踪到正文位置。找不到对应位置的，继续补写。

## review_notes

推荐文件：

```text
review_notes/
├── uncertainties.md
├── confusion-audit.md
└── source-audit.md
```

记录格式：

```markdown
- [ ] `sources/ch01.pdf` p.12：分段函数第二行参数可能是 `b`，OCR 识别为 `6`。
- [ ] `sources/exam-2024.pdf` p.3：第 4 题图中箭头方向不清，需要人工确认。
- [x] `sources/slides-02.pptx` slide 18：定理条件已与教材核对，正文已修正。
```

正文中不要插入大量“可能”“疑似 OCR 错误”等提示；保持讲义连贯，把核对项独立保存。

## Source Fusion

- 最终讲义不显式区分“课件已有”“参考资料补充”“模型补充”。
- 可以删去重复、松散、无教学价值的残片。
- 对跳步处补推导，对概念断裂处补连接，对高频考点补典型例题。
- 若来源之间冲突，优先采用用户指定教材或往年题口径，并在 `review_notes/` 标出冲突。
