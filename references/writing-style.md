# Writing Style Reference

读取本文件用于生成、润色或验收讲义正文。目标是让成品接近中文理工科专业教材或教师正式讲义，而不是聊天式回答、OCR 摘要或模型生成的泛化说明。

## Style Goal

讲义应呈现为经过教师整理和编辑的学习文本：术语稳定，句子克制，段落有明确教学功能，推理从定义、条件、结论和例题自然展开。正文不解释“我将如何讲解”，也不展示生成过程。

## Positive Rules

- 用“定义、定理、命题、证明、说明、例题、解、注、推论、方法”组织内容，让每段承担明确功能。
- 叙述优先从对象、条件和结论出发。例如先说明研究对象，再说明所需假设，最后给出结论或方法。
- 概念解释要落在数学或物理事实上：符号含义、适用条件、等价表述、几何意义、边界情形、典型用法。
- 困惑点应作为正文组织的暗线自然出现：在读者容易接不上处补说明，在方法选择前交代触发特征，在容易混淆处用注记或反例澄清，而不是另起一串问答。
- 例题解析采用正式解题书写：先判断适用条件，再列式计算，最后回到题目要求。必要时用“因此”“于是”“由……可得”“这说明”等连接推理。
- 方法总结要可操作：说明触发特征、适用条件、计算步骤和常见失效情形，避免只写鼓励性提醒。
- 语言保持中性、凝练、可检查。可以平实，但不要像聊天记录。

## Avoid AI Flavor

正文中避免下列表达类型：

- 自我指涉：如“我们将会看到”“接下来我们来学习”“本文将帮助你”“本讲义旨在”。
- 泛化串场：如“需要注意的是”“总的来说”“综上所述”“通过以上内容可以发现”，除非后面紧跟具体数学结论。
- 口语鼓励：如“不要慌”“很简单”“只要记住”“你会发现”“非常重要”。
- 模板化价值判断：如“这是一个非常经典且重要的概念”“在实际应用中具有广泛意义”，除非立即说明具体应用或考试触发点。
- 过度列点：连续使用“首先、其次、最后”而缺少数学推理；把正文写成提示清单。
- 困惑点标签化：如“学生困惑一”“你可能会问”“常见疑问是”，除非该位置本来就是审计记录而非讲义正文。
- 来源暴露：如“根据课件”“模型补充”“OCR 显示”“参考资料中提到”。来源差异和不确定性写入 `review_notes/`。

## Rewrite Patterns

- 把“下面我们来看连续性的定义”改为“连续性刻画函数值在自变量微小变化下的稳定性。其局部定义如下。”
- 把“需要注意的是，介值定理要求函数连续”改为“介值定理的关键条件是闭区间上的连续性；若区间内存在间断点，端点异号并不能保证零点存在。”
- 把“这个方法非常常用，考试经常考”改为“分段函数参数题通常以分界点为检查对象，计算左右极限并与函数值比较。”
- 把“综上所述，我们可以得到答案”改为“故连续性的充要条件为……”
- 把“学生可能困惑为什么要闭区间连续”改为“闭区间连续性保证函数值在端点之间没有跳跃；若区间内允许间断，端点函数值的符号关系不能控制中间取值。”

## Final Style Pass

定稿前逐段检查：

1. 每段是否有明确功能，是否能归入定义、解释、推导、例题、注记或方法。
2. 是否存在空泛评价、寒暄式引导、生成过程说明或来源暴露。
3. 是否有“显然”“不难看出”等跳步表达；若有，补出所用定义或运算理由。
4. 是否为了显得完整而堆叠套话；若是，删去套话，补具体条件、公式或例子。
5. 术语、符号和句式是否与同一讲义中其他章节一致。
6. 困惑点是否已经转化为正文中的启发性解释，而不是以清单或问答形式贴在段落之外。
