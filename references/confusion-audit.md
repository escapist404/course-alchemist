# Confusion Audit Reference

读取本文件用于生成或更新讲义、例题解析、题库讲解和增量章节。目标是把“系统性”落实为可检查的困惑点覆盖，而不是只让内容看起来完整。

## Core Principle

掌握知识的人常会低估基础断点：他们觉得显然的定义展开、符号含义、条件作用、代数变形、图形直觉和方法触发点，往往正是学生卡住的位置。生成讲义时必须主动扮演初学者，先找卡点，再写正文。

## Audit Procedure

每个章节或主要概念块按下面顺序处理：

1. 拆成最小知识单元：定义、对象、符号、条件、结论、方法、典型题型、前置技能。
2. 为每个知识单元列出学生可能追问的“为什么/怎么来/什么时候能用/哪里会错”。
3. 判断原资料是否只是一笔带过：若只给公式、结论、图或例题答案，需要补解释。
4. 将每个困惑点映射到讲义位置：定义解释、推导、图示、例题步骤、易错点、练习或题后反思。
5. 对未覆盖项立刻回填正文；若必须依赖用户确认或源材料不清，写入 `review_notes/confusion-audit.md`。
6. 完成后做一次反向检查：假设学生只读本讲义，不看原课件，是否能回答困惑清单中的问题。

## Confusion Types

至少检查这些类型：

- 概念入口：这个对象是什么，和相近概念有什么差别，为什么要定义它。
- 前置知识：读懂本段需要哪些更基础事实，讲义是否已经提醒或补齐。
- 条件作用：定理或公式的每个条件在防止什么失败；去掉条件会怎样。
- 符号和对象：变量、集合、区间、方向、单位、坐标、下标、映射对象是否明确。
- 推导跳步：等式变形、极限交换、积分换元、矩阵运算、向量方向、符号变化是否有中间理由。
- 方法触发：题目中哪些词、结构或数据提示应使用这个工具。
- 反例边界：容易误用的场景、定理不适用的边界、常见错误判断。
- 图形直觉：空间关系、方向、区域、路径、力、场、变换是否能不用脑补就看懂。
- 计算细节：容易算错的符号、端点、常数、单位、定义域、特殊值。
- 迁移练习：学生能否从一个例题迁移到相邻题型，而不是只记住答案。

## Writing Rules

- 困惑点应进入正文教学结构，而不是只堆在“易错点”盒子里。定义、推导和例题本身要变得更可学。
- 不要用“显然”“容易得到”“不难看出”跳过关键步骤；若必须压缩，至少说明所用事实。
- 每个核心定理至少解释一次条件为何存在，并给一个误用提醒或边界例子。
- 每个核心方法至少配一个完整例题，例题解答要写出选择方法的原因。
- 对强空间直觉内容，困惑审计和图示补强检查必须合并执行。
- 若内容面向冲刺训练，可以压缩背景，但不能省略方法触发点、适用条件和高频错误。

## Review Notes

推荐文件：

```text
review_notes/
└── confusion-audit.md
```

记录格式：

```markdown
# Confusion Audit

## 第 2 章 导数

- [x] `tex/chapters/ch02.tex`：已补充导数定义中“自变量增量趋于 0”与“函数增量比值”的含义。
- [x] `tex/chapters/ch02.tex`：已在例题 2.3 说明为什么先检查可导再使用切线公式。
- [ ] `sources/slides-04.pdf` p.18：课件中的参数方程速度方向图不清，需确认箭头方向后补图示。
```

## Acceptance Checklist

交付前至少确认：

- 每个主要概念块都有困惑点清单或等价的覆盖说明。
- 每个清单项能映射到讲义正文、例题、图示、易错点或练习。
- 没有把关键基础解释留给读者自行脑补。
- 仍不确定的困惑点已进入 `review_notes/confusion-audit.md`，而不是混入正文。
